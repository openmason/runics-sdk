# Repository Structure

This document provides an overview of the Runics SDK repository structure.

## Root Files

### Configuration
- `package.json` - Root workspace configuration
- `pnpm-workspace.yaml` - pnpm workspace definition
- `turbo.json` - Turborepo build orchestration
- `tsconfig.base.json` - Shared TypeScript configuration
- `biome.json` - Biome linter/formatter configuration
- `.gitignore` - Git ignore rules

### Documentation
- `README.md` - Project overview and quick start
- `CHANGELOG.md` - Release history and changes
- `CONTRIBUTING.md` - Contribution guidelines
- `SECURITY.md` - Security policy and vulnerability reporting
- `LICENSE` - MIT License
- `RELEASE.md` - Release process guide
- `REPOSITORY.md` - This file

### CI/CD
- `.github/workflows/ci.yml` - Continuous integration (lint, test, build)
- `.github/workflows/publish.yml` - Automated npm publishing

## Packages

### `packages/client/` - runics-client

TypeScript SDK for the Runics API.

**Key Files:**
- `src/client.ts` - Main RunicsClient class
- `src/schemas.ts` - Zod validation schemas
- `src/types.ts` - TypeScript type definitions
- `src/errors.ts` - Error hierarchy
- `src/mcp.ts` - MCP server factory
- `src/index.ts` - Package exports

**Configuration:**
- `package.json` - Package metadata and dependencies
- `tsconfig.json` - TypeScript config (extends base)
- `tsup.config.ts` - Build configuration (ESM + CJS)
- `.npmignore` - Files excluded from npm package

**Tests:**
- `src/__tests__/client.test.ts` - Client method tests
- `src/__tests__/schemas.test.ts` - Schema validation tests
- `src/__tests__/errors.test.ts` - Error handling tests

### `packages/cli/` - runics

Command-line interface for Runics.

**Key Files:**
- `src/index.ts` - CLI entry point
- `src/commands/` - Command implementations (search, inspect, feedback, etc.)
- `src/formatters/` - Output formatters (table, JSON, composition)
- `src/config.ts` - Configuration resolution

**Configuration:**
- `package.json` - Package metadata, bin entry, dependencies
- `tsconfig.json` - TypeScript config (extends base)
- `tsup.config.ts` - Build configuration (ESM with shebang)
- `.npmignore` - Files excluded from npm package

**Tests:**
- `src/__tests__/config.test.ts` - Config resolution tests
- `src/__tests__/formatters/` - Formatter tests

## Build Artifacts (Ignored)

These are generated by build commands and ignored by git:

- `dist/` - Compiled JavaScript and type definitions
- `node_modules/` - npm dependencies
- `.turbo/` - Turborepo cache
- `*.tsbuildinfo` - TypeScript incremental build info

## Development

### Build
```bash
pnpm build        # Build all packages
```

### Test
```bash
pnpm test         # Run all tests
```

### Lint
```bash
pnpm lint         # Check code style
pnpm lint:fix     # Fix code style
```

## Package Distribution

When published to npm, each package includes only:
- `dist/` - Compiled code and types
- `README.md` - Package documentation
- `package.json` - Package metadata
- `LICENSE` - MIT License (inherited)

Source files, tests, and configuration are excluded via `.npmignore`.
